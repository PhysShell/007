# Consilium Web / Server / UI — Assessment for the 007 Cockpit

Sources: `core/src/server.rs` (Axum + WS), `core/src/protocol.rs` (wire types), `core/src/event.rs`, `ui/src/**` (React/Vite), `desktop/src-tauri/**` (Tauri), tests in `core/tests/server_test.rs`.

## Server / WebSocket findings

- **Axum routes** (`server.rs:105-113`): `GET /ws/session` (WebSocket) + four REST reads — `/api/quota`, `/api/doctor`, `/api/config`, `/api/version` (`server.rs:183-224`). REST handlers are pure/stateless snapshots.
- **WS session lifecycle is one-run-per-socket** (`server.rs:267-431`). The **first text frame *is* the run**: parsed as a `SessionRequest`, and for `Conduct` the run is spawned inline (`server.rs:335-380`). Control-only first frames (`Cancel`/`Pause`/`Resume`/`Interject`) are rejected — "the first frame must describe a run" (`server.rs:311-334`). The socket then streams events and ends with exactly one terminal frame, then `Close` (`server.rs:413-431`).
- **Origin validation** (`server.rs:140-181`): upgrades refused unless `Origin` is absent (non-browser) or loopback (`localhost`/`127.0.0.1`/`::1`) or Tauri webview (`tauri.localhost`). Framed as CSRF/DNS-rebind defense; suffix-attack cases tested (`server.rs:493-524`). **No auth token — origin is the only gate.**
- **cwd confinement** (`server.rs:336-346`): client `cwd` (default = launch root) must pass `cwd_within_root` before a run starts.
- **Active-run semaphore = 1** (`server.rs:55,70,349-357`): a second `/ws/session` while a run is active is refused, starts no run (test `server_test.rs:364-408`). One run per server, period.
- **Event forwarding / backpressure** (`server.rs:253-304`): the run executes inside a task-local `PROGRESS_SINK.scope(...)`; a `WsSink` serializes each `AgentEvent` into an **unbounded** mpsc channel drained by a writer task, so the engine never blocks on socket backpressure (`server.rs:8-10,256-265,297-304`). Unbounded = a slow reader grows memory unbounded; the *bounded* backpressure (channel of 256) lives between adapter and orchestrator (`sessions.rs:89-91`), not at the socket.
- **Cancellation / browser-close = run death (the key divergence).** The handler `select!`s the run task against the socket receiver; **any of** a `{"kind":"cancel"}` frame, `Close`, socket error, or EOF calls `run.abort()` and awaits it so children are killed (`server.rs:382-411`, esp. `403-407`). Aborting drops the session; `sessions.rs` spawns children with `kill_on_drop(true)` + abort-on-drop `ReaderTask` (`sessions.rs:32-36,94-100`), SIGKILLing agents. Comment: "a closed browser tab cannot keep burning quota" (`server.rs:360-362`); proven by `server_test.rs:318-363` (worker marker never written). **→ Closing the browser stops the agent — exactly what 007 forbids.**
- **Reconnect / persistence / cursor / resume — NONE.** `server.rs` never writes a transcript or event log. Live events are ephemeral — streamed once and gone. Whole-run transcripts are saved only by the **CLI** subcommands (`main.rs:447-458,514-525,584-630`), not the server. No `seq`/cursor, no `?since=`, no replay, no reconnect anywhere in `core/src` or `ui/src`.
- **Multiple conversations — unsupported.** Semaphore(1) + one-run-per-socket = a single active run per server process (per Tauri workspace).
- **UI state ownership** (`ui/src/session/reducer.ts`): all run state lives in an in-memory reducer (`reducer.ts:12-45`). The events array is an "append-only live log" *within a run* (`reducer.ts:16,116-139`), but `start`/`reset` resets it (`reducer.ts:60-61,84`), and a page reload loses everything. `useSession` owns the single socket, one per run, torn down on unmount/reset with a generation counter guarding stale sockets (`useSession.ts:23-137`). The UI **initiates** the run by opening the socket and sending the `conduct` frame itself (`useSession.ts:49-83`, `StartRunForm.tsx:28-37`) — **the UI launches the agent directly**, another 007 divergence.
- **Quota display** (`ui/src/dashboard/useQuota.ts`): independent of the WS — polls `GET /api/quota` every 10s while the Usage tab is active, abort-on-unmount (`useQuota.ts:17-49`). Survives run/socket lifecycle. Clean and reusable.
- **Desktop / Tauri coupling** (`desktop/src-tauri/src/server_lifecycle.rs`): Tauri embeds the server **in-process**, binds `127.0.0.1:0`, ties one server to one user-picked workspace as `launch_root` (`server_lifecycle.rs:58-126`). Restart aborts the previous server task (`:60-66`). UI discovers the URL via Tauri IPC `get_server_state` (`commands.rs:24-44`, `runtime.ts:43-83`); on launch auto-starts the last saved workspace (`lib.rs:40-59`). Server lifetime is bound to the desktop app process.

## The five web answers

**1. What can be reused in the 007 Cockpit?** Much of the presentation/typing layer, little of the transport layer. Reuse directly: the React presentational components (`EventStream`, `EventRow`, `ResultPanel`, `SessionHeader`, `UsageBadge`, `QuotaDashboard`, `TableView`, `StatusPill`) — plain props, never touch the socket (`App.tsx:15-65`). The **pure reducer fold** (`reducer.ts:58-139`) + append-only event model. The **tolerant frame parser** (`parseFrame.ts:13-29`) forwarding unknown `type` tags so new server variants degrade gracefully. The **ts-rs single-source-of-truth codegen** (Rust types in `protocol.rs`/`event.rs` export to `ui/src/protocol/`, `protocol.rs:11-16,29`) with disjoint-tag guard test (`protocol.rs:311-322`). The REST **quota poll hook** (`useQuota.ts`). The **origin-validation** and **cwd-confinement** helpers. The **runtime-resolution seam** (`runtime.ts:77-96`) for pointing the Cockpit at a local or remote o7d.

**2. What is hard-wired to a ONE-RUN WebSocket lifecycle?** The `active_run` **Semaphore(1)** + its refusal (`server.rs:55,70,349-357`). **"First frame = the run"** (`server.rs:335-380`); control frames only valid after a run exists (`:311-334`). The run task **living inside the socket handler**, `select!`ed against the socket so socket-close aborts it (`:374-411`). **No run id** anywhere in `ServerFrame`/`SessionRequest` (`protocol.rs:27-112`) — a socket can only refer to "the run on this socket." Terminal-frame-then-`Close` finality (`:413-431`). Client-side `useSession` opening one socket per `start()`, torn down on `reset`/unmount (`useSession.ts:31-83,131-134`).

**3. What must be rewritten for an APPEND-ONLY event ledger with cursor resume?** Essentially all of `handle_session` (`server.rs:267-431`) and the run-ownership model: **Decouple run lifetime from the socket** — the run must live in o7d, created by an authenticated command (UI never launches agents directly), not "the first WS frame." **Persist every event to an append-only ledger** with a monotonic per-run `seq`, keyed by `run_id`/`conversation_id`; today events go only into an ephemeral unbounded channel (`:256-304`). **Make the socket a cursor-based subscriber** (`?run_id=…&since=<seq>`; replay from cursor, then tail). **Remove semaphore-of-1 / add multi-run + multi-conversation** registry. **Invert cancellation**: socket close must **not** `run.abort()` (`:403-407`) — the single change most central to "closing the browser must not stop the agent"; cancellation becomes an explicit authorized command against a `run_id`. The kill-on-drop child model stays for *explicit* cancel/timeout, but the run future must be owned by o7d's registry, not a socket handler.

**4. Can the React components be kept independently of the wire protocol?** Yes — already enforced, the codebase's strongest asset for 007. The impure socket layer is isolated to a single hook, `useSession` ("the only impure layer", `useSession.ts:21-23`); everything downstream is a **pure fold** (`reducer.ts:56-58`) feeding presentational components with plain props (`App.tsx:33-61`, `EventStream.tsx:9-27`). To move to a ledger/cursor backend, replace `useSession` with a cursor-subscribing client that dispatches the same `SessionAction`s; keep the reducer, components, `parseFrame`, generated protocol types unchanged. One caveat: the reducer's append-only array assumes strictly-increasing arrival (`EventStream.tsx:20-21` index keys "safe because append-only") — with cursor replay you must dedupe/order by `seq`, a small reducer change.

**5. Is there real value in the desktop/Tauri shell for an ANDROID-FIRST 007?** Little, and its lifecycle model is actively wrong for the target. The shell embeds the server **in-process and binds server lifetime to the app process** (`server_lifecycle.rs:58-126`, `lib.rs:13-59`). On Android-first, where the agent must survive backgrounding/kill, an in-process localhost server whose run dies on socket/app teardown is the anti-pattern 007 avoids. The Android scaffolding exists (mobile entry `lib.rs:12`, `desktop/src-tauri/icons/android/**`), but the *architecture* — localhost in-process server + app-close-kills-run — does not translate. Correct shape: o7d as a **standalone/remote service** owning runs+ledger, with the client (Tauri/native/browser) as an authenticated **viewer/controller** subscribing by `run_id`. Salvage icon/asset scaffolding and the `runtime.ts` backend-resolution seam; drop the in-process lifecycle coupling.

## Reuse verdict per layer

| Layer | Verdict | Rationale |
|---|---|---|
| **Axum server** (`core/src/server.rs`) | **PARTIAL — keep scaffolding, rewrite `handle_session`.** | Router, four REST snapshot handlers (`:183-224`), `origin_allowed` (`:140-163`), `cwd_within_root` reusable as-is. The run-lifecycle core (`:267-431`) — first-frame-is-run, semaphore(1), run-tied-to-socket, socket-close-aborts — must be replaced by a run registry + append-only ledger + cursor subscription. |
| **WS protocol** (`protocol.rs`, `event.rs`) | **KEEP the shape, EXTEND the frames.** | Typed frames, snake_case tag discipline, disjoint-tag guarantee (`protocol.rs:311-322`), ts-rs codegen — an excellent typed-wire foundation. Add `run_id`, `conversation_id`, monotonic `seq`/cursor, subscribe-vs-create split (socket never *starts* a run). |
| **React components** (`ui/src/**`) | **KEEP.** | Pure reducer + presentational components already wire-agnostic; only `useSession` is swapped for a cursor/ledger client. Reducer needs a minor `seq`-ordering/dedupe tweak for replay. Highest-leverage reuse in the codebase. |
| **Tauri shell** (`desktop/src-tauri/**`) | **DROP for android-first (demote to a viewer).** | In-process server + lifetime-bound-to-app is the wrong model for an agent that must outlive the client. Salvage icon/asset scaffolding + the `runtime.ts` seam; discard the server-lifecycle coupling. |

**Bottom line on the 007 invariant:** Consilium's server **does** tie run lifetime to the socket — the run task is owned by the WS handler and `run.abort()` fires on socket close/error/EOF (`server.rs:382-411`), killing agent children via `kill_on_drop` (`sessions.rs:94-100`). A deliberate anti-quota-leak feature there, but a direct divergence from 007's "closing the browser must not stop the agent" and "only o7d is source of truth." The server keeps **no** durable per-event state (no ledger, cursor, reconnect); durable state exists only as whole-run transcript JSON written by the CLI paths (`main.rs:447-630`), which is also what MCP's `search_recall`/`page_in` read — so the live UI and the recall store are already two disconnected persistence stories that 007's single append-only ledger would unify.
