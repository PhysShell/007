# consilium-module-map.md

## Intro

`consilium` (crate at `/home/physshell/Documents/repos/consilium/core`, ~21.3k LoC of `src` + ~8.9k LoC of `tests`) is a **multi-agent orchestrator for subscription CLI agents**. It never calls a model API directly — it *forks vendor CLIs* (`claude`, `codex`, `agy`, `grok`) as child processes, normalizes their stdout into one `AgentEvent` stream, and drives them through a conductor/worker deliberation loop. Two front doors expose the engine: a **localhost WebSocket server** (`server.rs`, for the desktop UI) and a **stdio MCP server** (`mcp.rs`, "attached-conductor" mode where the user's own Claude Code session is the conductor). A CLI (`main.rs`) offers `conduct` / `council` / `review` / `auto` / `serve` / `mcp` / `doctor` / `init` / `models` / `eval`.

**Five architecture facts that dominate the 007 read:**

1. **The only decider of "shipped" is an LLM conductor, backed by a mechanical build/test *grounding veto*.** An agent's own "DONE" is never acceptance (`conduct.rs:669-680`, `eval.rs:328-333`). Closest analog to o7d's "only o7d decides verdict" — but Consilium's judge is itself a model, not a trusted deterministic plane.
2. **Silent model failover is the design center, not forbidden.** Every model call routes through `resilience::run_with_failover` over a `RoleConfig.ladder()` (`resilience.rs:190`, `config.rs:43`). There is **no `ModelPolicy`/`Exact` type anywhere** — the inverse of the 007 invariant. (Failover is at least *loud*: stderr + a recorded `Fallback` list.)
3. **The "sandbox" story is exactly what 007 warns against — and it's unwired.** The TOCTOU-hardened *git worktree* isolation, the *trust store*, and the *safety preflight* (`safety/git.rs`, `safety/trust.rs`, `safety/preflight.rs`) are built and heavily tested but have **NO production callers**. Live runs execute workers directly in a **shared cwd** (`conduct.rs:328-332`). A worktree is not a sandbox, and here it isn't even in the path.
4. **The run "ledger" is a post-hoc whole-file JSON, not an append-only event ledger.** `run_conduct` assembles a `transcript: serde_json::Value` in memory and the *caller* writes it once at the end (`conduct.rs:934`, `main.rs:630`). No crash/reconnect recovery. The only append-only SQLite is `quota.rs`'s `usage_log` — but that's token counts, not run events.
5. **Live control is boundary-based, not mid-flight.** `operator.rs` pauses/resumes/interjects a run only at the *top of the next subtask dispatch* (`conduct.rs:364`), never mid-worker-call. Not permission-mode live-switching of a running worker.

The confinement primitive that *is* wired everywhere untrusted input picks a path is `confine::cwd_within_root` (a canonicalized path-jail), used by `server.rs`, `mcp.rs`, `preflight.rs`.

## `event.rs` — the normalized event contract
**Purpose.** `Provider` (`event.rs:6`: Claude/Codex/Gemini/Grok) and `AgentEvent` (`event.rs:44`), the single normalized event enum every adapter maps its CLI output into. Variants: `SessionStarted`, `Thinking`, `Message`, `ToolCall`, `FileChanged`, `Usage{input,output}`, `Completed{result}`, `Failed{error}`, + operator `Paused`/`Resumed`/`OperatorNote` (`event.rs:77-86`). **Public API.** `enum Provider`(:6), `as_str`(:17), `FromStr`(:27), `enum AgentEvent`(:44); serde tagged `type` snake_case, `#[ts(export)]`. **Deps.** `serde`, `ts_rs`; depended-on universally. **State.** None — pure value type; events live in an mpsc channel, never persisted as events. **Failure.** Infallible; a test pins u64 tokens serialize as JSON number not bigint (:130-139). **Coverage.** In-module (:89-140), zero-quota. **007.** *adopt-candidate* — clean normalized worker-event contract; but ephemeral (channel), so an append-only ledger must persist these (Consilium does not).

## `adapters/` — vendor-CLI process adapters
**Purpose.** One `Adapter` per vendor CLI: build argv + parse stdout into `AgentEvent`s. Parsing is pure/no-I/O (fixture-testable, `mod.rs:46`). **Public API.** `mod.rs`: `RunRequest{prompt,model,cwd,advisory,write}`(:10-32); `FailureKind{ModelUnavailable,RateLimited,Transient}`(:35); `trait Adapter`(:48). **No `PermissionMode` enum** — permission = raw CLI flags gated on `write`. Claude(`claude.rs:5`): `-p <prompt> --output-format stream-json --verbose`; write→`--permission-mode acceptEdits`; sums disjoint cache tokens into input(:85-87). Codex(`codex.rs:5`): `codex exec --json`; advisory→`--skip-git-repo-check`; write→`--sandbox workspace-write`; cached is subset(no sum). Gemini(`gemini.rs:15`): `agy`, non-streaming `parse_final`; write→`--dangerously-skip-permissions`; emits **no `Usage`**; empty→fail-closed. Grok(`grok.rs:31`, beta): `grok --output-format streaming-json`. **Deps.** `tokio::process`, `serde_json`, `event`; adapter factories live outside the dir (`roles.rs:12`, `doctor.rs:179`, `main.rs:352`). **State.** Stateless unit structs (`Arc`-shared). **Trust.** Parses untrusted vendor stdout, chooses privilege flags — but **no adapter reads env/secrets in production** (all `std::env`/`fs` are `#[cfg(test)]`); the child CLI inherits parent env implicitly. **Failure.** **Fail-open on parse:** garbage → empty Vec, unknown types dropped. Real exit/stderr failures synthesized by `sessions.rs`. **Coverage.** In-module parse tests over `tests/fixtures/{claude,codex,gemini,grok}/*.jsonl` incl. `recorded/` and `garbage_lines.jsonl`. Zero-quota. **007.** *adapt* — the `Adapter` trait + `FailureKind` is a good porting seam for a Claude SDK worker; the two-bool permission model + fail-open parsing need hardening under an untrusted-content threat model.

## `sessions.rs` — process lifecycle & event streaming
**Purpose.** `spawn(adapter,req)` forks the CLI child, streams normalized events over a bounded channel, drains stderr concurrently, SIGKILLs the child when the run is dropped/aborted. **API.** `SessionHandle{id, events: mpsc::Receiver<AgentEvent>, task: ReaderTask}`(:9); `ReaderTask`(aborts child on drop, :23-36); `spawn(...)`(:76). **Deps.** `tokio`, `adapters`, `event`. **State.** Owns child + reader/stderr tasks; nothing persisted; bounded 256-slot channel(:91) = cooperative backpressure onto stdout. **Trust.** **The fork point** (:102), `stdin(null)`, piped stdout/stderr, `kill_on_drop(true)`(:100). **Hard `bail!` if `advisory && write`**(:84-88) — real guard in release, not debug_assert. **Failure.** Spawn err→Err; non-zero exit→synthesized `Failed` with 4 KiB stderr tail(:134-149); receiver-dropped→SIGKILL child(:117-122); double-terminal not deduped, first wins(:70-72). **Coverage.** `tests/sessions_test.rs` — **real process spawning** (`sh` scripts: crash, stderr, missing-binary, advisory+write bail). No vendor CLI/network. **007.** *adopt-candidate* — kill_on_drop → dropped future kills child, concurrent stderr drain, bounded backpressure = exactly what a long-lived Claude SDK worker needs. Note: `kill_on_drop` is process death, not a sandbox; pair with Sandboy.

## `orchestrator/` — the deliberation engine
`mod.rs`(:1-18) declares 19 submodules. Choke point: `resilience::run_with_failover` — **every** conductor/worker/supervisor/reviewer/arbiter/council call flows through it → `runner::run_to_completion` → `sessions::spawn`.

### `runner.rs` — one session to completion
`RunStatus{Completed,Failed,TimedOut}`(:10); `RunOutcome{session_id,final_text,events,status}`(:17); `run_to_completion(adapter,req,quota,timeout)`(:34). No persistence except side-channel quota write(:66, failure logged not fatal). **First terminal `Completed` authoritative**(:73-76); timeout aborts+awaits reader → child SIGKILLed before return(:96-113); no-usage completed run gets heuristic token estimate(:127-141, the Gemini path); live-taps each event to `progress::emit`(:85). Coverage `tests/runner_test.rs` zero-quota. **007.** *adopt-candidate* — clean "drive one worker, timeout kills child, first-terminal-wins."

### `resilience.rs` — the failover engine (007-critical)
`Rung{candidate,adapter}`(:14); `RetryConfig`(:25, `prod()`=2s base,3 retries,cold_after 3,:52); `ModelHealth`(:76, per-run `Arc<Mutex<HashSet>>` dead/cold); `Fallback{from,to,reason}`(:141); `FailoverResult{outcome,rung_used,fallbacks,model_used}`(:148); `run_with_failover(...)`(:190). Walks rungs(:205); demotes to `ladder.get(i+1)` on failure(:298-305); marks dead **only** on `ModelUnavailable`(:284); `RateLimited`/`Transient`/spawn-`Err` retry same rung then demote(:251-270); rate-limit breaker trips rung "cold"(:286). **Loud**: every demotion → stderr(:304) + recorded `fallbacks`. In-memory per run; **fail-closed at exhaustion** (`bail!` empty :198 / all-failed :313); per-rung failures fail-*forward*, never spurious success. Coverage `tests/resilience_test.rs` (scripted `SequencedAdapter`/`SpawnFailAdapter`) zero-quota. Tight to `runner`,`adapters::FailureKind`,`config::ModelCandidate`,`quota`. **007.** **reject** for a `ModelPolicy::Exact` world — this module *is* silent(ish) model switching, the antithesis of "no failover." *reference-only*: its loud stderr + `Fallback` records are a good model for making a switch auditable if o7d ever permits a bounded, logged fallback. To port Exact: run a **single-rung ladder** (`config.rs:315` shows 1-rung is legal) and treat any demotion as a hard run failure.

### `conduct.rs` — the conductor/worker engine (1999 lines)
Decompose task → subtask DAG → per subtask route→work→capture→verify→supervise→evaluate→review→arbiter, with replan + wall-clock budget, one JSON transcript. API: `Subtask`(:21), `Plan`/`parse_plan`(:36/:40); `EvalDecision{Accept,Rework,Fail}`(:45); `SupervisorStatus{Ok,Concern,Halt}`(:78); `ArbiterDecision{Ship,Fail}`(:130); `RoleHandle{ladder}`(:169); `ConductDeps`(:173); `ConductOutcome{completed,halted,failed,transcript}`(:201); `MAX_REWORKS=2`(:212); `run_conduct(...)`(:226); private `GateDecision{Accept,Ship,Rework,Fail}`(:1014), `run_review_gate`(:1056).
**Control flow:** (1) Decompose — conductor `advisory:true,write:false`(:269-291); non-empty plan or error(:293). (2) Outer `loop`(replan) → `topology::plan_waves`(:347) → `'plan for wave` → `'next_subtask`(:354-355). (3) `operator::checkpoint().await`(:364) — **the only park point**. (4) Budget gate → `break 'plan`(:370-381); unmet-deps → skip (cascades). (5) Attempt loop `0..=MAX_REWORKS`(:437): `pick_worker_by_provider`(:439) → worker `run_with_failover` **write:true**(:449) → `capture_changes`(:515, infra fault propagates `?`) → `run_verify`(:521) → stagnation fingerprint(:527) → supervisor gate(:538, `Halt`→global abort) → conductor evaluation(:626) → **grounding rule**(:669-680) → dispatch(:704).
**Verdict/acceptance (007-critical).** A subtask is `completed` only when all agree: (1) conductor LLM returns `Accept`(:704); (2) **grounding**: if a verifier ran & tests failed, `Accept` mechanically rewritten to `Rework`(:669-680) — a non-LLM veto; (3) reviewer has no `Critical`(:1112); (4) arbiter (only on rework-exhaustion) returns `Ship`(:1196). **Agent "DONE" is never acceptance** — worker text is merely an input to the eval prompt(:627-631). **State.** Entirely in-memory locals; no DB/files; durable output is the returned `transcript` JSON, persisted by caller(`main.rs:630`); working memory re-derived per call. **Trust.** Spawns provider CLIs transitively; workers **mutate the shared cwd**(write:true); reads repo via `capture_changes`; executes `sh -c` build/test via `run_verify`. **No worktree isolation** — subtasks share one cwd sequentially(:328-332). **Failure.** Unparseable model output → fail-safe (eval→Rework:657, supervisor→Concern:570, arbiter→Fail:1192, review-None→blocks:1114); infra-unavailable → advisory gates **fail-open** (supervisor:602, reviewer:1102), but **arbiter fails closed**(:1216), conductor eval failing is hard `?` abort(:653); `failed` first-wins. **Coverage.** `tests/conduct_test.rs` (**3953 lines**, largest suite), scripted ladders, zero-quota; exercises grounding veto, Halt, review/arbiter gates, replan, budget, stagnation. **Coupling.** Tightest node — pulls resilience,runner,sessions,routing,topology,stagnation,changes,verify,review,operator,progress,prompts,roles,council,quota,config,event. Not extractable without the whole orchestrator. **007.** *adapt* — the **grounding rule** (build/test veto over agent opinion) is the single most portable idea and directly serves "DONE != acceptance." But the *decider* is an LLM conductor, not a trusted deterministic plane; to match o7d, relocate the accept/fail verdict to o7d and keep conduct as a worker-driver only.

### `operator.rs` — live control (park/resume/interject)
`OperatorHandle`(:61, pause/resume/interject); `task_local! OPERATOR_CONTROLS`(:132); `checkpoint() -> Vec<String>`(:145). Pause = `watch::Sender<bool>`(level-triggered; `_paused_rx` keeps receiver-count>0 so a pause/resume race between checkpoints isn't dropped, :41-51); `checkpoint` no-ops instantly when no handle installed. Installed at `server.rs:374`; frames routed at `server.rs:393-395`. In-memory `Arc<Inner>`, no persistence/spawn/repo access; sends ignore Err. Coverage in-module(:161-288) zero-quota. Loose (task-local). **007.** *adapt* — task-local ambient-control + level-triggered `watch<bool>` is a clean way to inject live control into a long-lived worker. **But it parks between subtasks, not mid-call** — *not* the "permission-mode live switching for a running Claude SDK worker" that 007 wants; you'd need an in-worker control channel that flips permission mode without ending the current turn.

### `council.rs` — advisory multi-agent Q&A
Separate entry point (not part of run_conduct): 3-stage anonymized council — parallel independent answers → blind peer scoring → chairman synthesis. **All calls advisory:true, write:false.** `CouncilMember{label,ladder}`(:9, reused by `ConductDeps.workers`); `CouncilOutcome{synthesis,answers,failed_members,transcript}`(:35); `run_council(...)`(:43). No pass/fail — synthesized answer, no grounding; in-memory (caller persists `main.rs:458`); read-only; fail-open per member; hard-fails only if zero answer(:100) or chairman fails(:180). Coverage `tests/council_test.rs` scripted zero-quota. **007.** *reference-only* — deliberation/second-opinion primitive, not a verdict authority.

### `eval.rs` — benchmark harness (the honesty keystone)
Run each task × approach (`Solo`/`Conduct`/`ConductNoGrounding`/`ConductCrossFamily`) in an isolated copy of a starter repo, score the tree with an **independent external `run_verify`**. `Approach`(:31); `EvalTask{...protected_paths}`(:94); `trait EvalDeps`(:115, injection seam); `TrialResult{success,pipeline_ok,...}`(:148); `dry_run_plan`(:245, "DRY RUN — no quota"); `run_suite`(:269). **Control flow (007-critical).** `run_trial`(:304): copy starter to tempdir + `git_init_commit`(:313-315); **in-memory** quota(:318, never touches real `usage.db`); run approach; **`restore_protected`** undoes tampering with scored test files via `git checkout HEAD -- <path>`(:326, so an approach can't pass by rewriting its own test); then **external oracle** `run_verify`, `success = v.ran && v.passed`(:332-333). The approach's own `pipeline_ok` is recorded but is **not** the score; not-run verify is **not** a pass. Per-trial tempdir + in-memory quota isolation; spawns git/verify/CLIs; an approach error never aborts the suite. Coverage `tests/eval_test.rs` scripted zero-quota; external-oracle rule tested end-to-end. **007.** **adopt-candidate** — strongest match to "agent DONE != acceptance / external authority decides": external, tamper-proofed (protected-path restore), independent scoring; per-trial tempdir isolation is a real (not kernel-level) isolation pattern; the oracle is deterministic build/test, exactly the signal o7d should own.

### `review.rs` — LLM diff-review gate
Send a diff to a reviewer, parse `{findings:[{severity,file,description}]}`, decide if `Critical` blocks. `Severity{Critical,Important,Minor}`(:13, lenient unknown→Minor:30); `Verdict{findings}`+`has_critical()`(:40-48); `ReviewResult{verdict:Option,raw_review,transcript,model_used}`(:58); `run_review`(:67)/`run_review_ladder`(:115, failover — the path conduct+MCP use). Gate: `review_blocks = verdict.map(has_critical).unwrap_or(true)` — **unparseable = fail-closed**(`conduct.rs:1112-1114`); only `Critical` blocks; transient reviewer failure degrades to Accept(`conduct.rs:1102-1108`); non-`Completed`→`bail!`(:90/142). Coverage `tests/review_test.rs` + in-module. **007.** *adapt* — cross-family review is a useful second gate; MCP exposes it as `review_diff` with explicit "`parse_ok:false`=fail closed" contract. Not a substitute for a deterministic verdict.

### `verify.rs` — build/test/lint grounding oracle
Resolve (config-wins-per-field, else ecosystem autodetect) + run the worktree's commands; return `VerifyOutcome{ran,passed,summary}`. Both conduct's internal accept-gate and eval's external scorer. `VerifyOutcome`(:7); `detect_commands`(:43, Cargo/npm/pytest/Makefile); `resolve_commands`(:63, via `safety::resolve_commands_with_provenance`); `run_verify`(:75)/`run_resolved_verify`(:82). No commands → `ran:false,passed:false` (**not-run is not a pass**); `blocking = label != "lint"`; each command `sh -c` with `kill_on_drop(true)` + per-command timeout (default 600s); timeout → SIGKILL + `TIMEOUT`(blocking); **lint advisory in every failure mode**(:96-158). **The executing arm** — arbitrary configured/detected shell in cwd; commands carry provenance (`UserProvided` vs autodetected) via `safety`. Coverage in-module + via conduct/eval; real `sh` in tempdirs. **007.** **adopt** — a deterministic, timeout-bounded, tamper-resistant grounding oracle is precisely the signal a trusted control plane should compute and own. Port it; move the *decision* to o7d.

## `transcript.rs` — the run "ledger" (session-store equivalent)
Human-readable JSON transcripts under `<base>/runs/<nanos>-<seq>-<kind>.json`. Explicitly **files, not SQLite**(:13-17: "transcripts are for humans to read and diff"). `SearchHit{id,kind,task,snippet}`(:5); `TranscriptStore{base}`(:16); `new`(:21), `default_base`(:25, `~/.consilium`), `save(kind,payload)`(:33), `search(query,limit)`(:55), `load_by_id`(:190). Whole-file JSON on disk, written **once at run end** by the caller(`main.rs:458/525/630/758`), not the engine; per-process atomic counter tiebreaks nanosecond collisions(:48); MCP `search_recall`/`page_in` read these back. `kind` filename-safety is a `debug_assert` only(:34). `save` propagates I/O Err; `search`/`load_by_id` lenient (skip unreadable, never panic). Coverage in-module(:239-375) zero-quota. **007.** **reject / reference-only** for the "append-only event ledger with crash/reconnect recovery" target: a *post-hoc snapshot*, not incremental — nothing written until run completes, so a crash mid-run loses the entire transcript, no reconnect/resume. To meet 007 you'd need per-event append + fsync + recovery reader. (The one append-only SQLite is `quota.rs`, but it logs tokens, not run events.)

## `server.rs` — localhost WebSocket server
Stream a run's `AgentEvent`s to a browser over `/ws/session`; also `GET /api/{quota,doctor,config,version}`. `ServerState`(:43); `router`(:105); `serve`/`serve_on`(:116/:131); `quota_snapshot`(:228). First WS frame = `SessionRequest`(:271). `Conduct` → confine `cwd`(:338) → `active_run` semaphore (**one run at a time**, :349) → spawn inside `OPERATOR_CONTROLS.scope(op, PROGRESS_SINK.scope(WsSink, run_conduct))`(:374-380). `select!`(:382) forwards later Pause/Resume/Interject/Cancel; **cancel or socket-close aborts the run task, dropped futures SIGKILL agent children**(:388-407). Terminal frame via `ServerFrame::from(&outcome)`(:415). Owns `launch_root`, shared `QuotaStore`, single-run semaphore. **Trust:** `origin_allowed` rejects cross-origin WS (CSRF/DNS-rebind, loopback + `tauri.localhost` only, :140-163); client `cwd` path-jailed to `launch_root`. Coverage `tests/server_test.rs` — real WS client, real git, asserts live+terminal frames; zero-quota. **007.** *adapt* — origin/loopback guard, single-run semaphore, cancel-kills-children, task-local sink+operator wiring are reusable; the cwd path-jail is *not* a sandbox.

## `mcp.rs` — attached-conductor MCP server (stdio)
Expose worker+quota+review+council+recall primitives as MCP tools so the user's live Claude Code session is the conductor. `McpServer`(:52)/`McpServerDeps`(:75); tools `run_worker`(:315), `quota_status`(:327), `review_diff`(:341), `council_run`(:356), `search_recall`(:367), `page_in`(:380); `serve_stdio`(:796). `run_worker_inner`(:477): confine cwd(:483) → resolve worker → `run_with_failover` **hard-coded `advisory:false, write:true`**(:517-527) → `capture_changes` (best-effort `.ok()` here, unlike conduct's `?`, :552) → `run_verify`. `review_diff`/`council_run` hard-coded `advisory:true,write:false`. Owns pre-resolved ladders, shared `QuotaStore`, `transcript_base`, `launch_root`. **Trust (documented :16-20):** the conductor is an LLM steered by untrusted repo content → **every cwd-taking tool path-jails cwd**; write/advisory can never combine at the tool boundary. Every tool returns structured `{ok,...,error}`, never panics; `ok:true` = "a rung ran," not "passed." Coverage `tests/mcp_test.rs` (875 lines) scripted zero-quota. **007.** *adapt* — "engine executes, external conductor decides" is close to "o7d decides, workers execute"; per-tool cwd confine + never-expose-advisory are good. But the decider is the user's LLM session, not a trusted plane; verdict authority not enforced server-side.

## `quota.rs` — SQLite token accounting (the only DB)
Append-only per-provider token log over a rolling 5h window; separates CLI-measured from heuristic estimates. `WINDOW_SECS=5*3600`(:9); `QuotaStore{conn:Mutex<Connection>}`(:24); `open`/`open_in_memory`(:29/:36); `record`/`record_at`/`record_estimated`(:69/:78/:92); `totals_since`/`estimated_totals_since`(:130/:148). **Owns a SQLite DB** at `~/.consilium/usage.db`; single table `usage_log(id,ts,provider,input_tokens,output_tokens,estimated)` — **INSERT-only (genuinely append-only)**, indexed on `(provider,ts)`; `Mutex<Connection>` makes it Sync for Arc sharing across MCP concurrent tools; tests use `open_in_memory`. Local file write only; no spawn/secret/socket; stores counts, never repo content. `anyhow::Result`; overflow guard(:109); negative-sum corruption → `bail!`(:137,159). **No spend ceiling / no zero-quota enforcement** — `config::QuotaConfig` fields exist but are *not consulted*; quota is pure accounting and never blocks a run (fail-open on spend). Coverage in-module(:166-230) zero-quota. **007.** *adopt-candidate* — append-only INSERT-only SQLite log with a windowed aggregate is exactly the storage shape an append-only ledger wants; extend schema from token rows to run-event rows + add crash-recovery reads. ## `safety/` — worktree isolation, trust store, command provenance (BUILT, TESTED, **UNWIRED**)
A fail-closed subsystem: owner-only state files, TOCTOU-hardened detached git worktrees, a repo+command-set trust store, a pre-session mode gate. **Critical: only `commands.rs` (`resolve_commands_with_provenance`,`VerificationCommand`,`CommandSource`) is consumed in production (by `verify.rs`). The worktree/trust/preflight crown jewels have NO production callers.**
- **`mod.rs`** — facade re-exports only(:7-20).
- **`commands.rs`** — `VerificationCommand{label,command,source,timeout_secs}`(:17), `CommandSource{AutoDetected,RepositoryConfig,UserProvided}`(:11), `resolve_commands_with_provenance`(:32), `digest_commands`(:27, SHA-256 — the trust anchor). *The only wired part.*
- **`fs.rs`** — `ensure_owner_only_dir`(:41), `write_owner_only_json`(:72): 0700 dirs / atomic 0600 writes via **rustix**, `NOFOLLOW`, symlink-refusing(:113-123). Fail-closed.
- **`git.rs`** — worktree engine: `inspect_repository`(:147), `create_detached_worktree`(:203), `remove_worktree`(:340), `reopen_prepared_worktree`(:373), `source_is_applyable`(:494). Manually materializes trees (no `git checkout`, so smudge/process filters + post-checkout hooks never run); fd+inode identity pinning detects directory substitution and **preserves both trees rather than deleting** on ambiguity. Aggressively fail-closed.
- **`preflight.rs`** — `inspect(PreflightInput) -> SafetyPreflightReport`(:106): confine cwd → inspect repo → resolve commands+digest → **mode matrix**(:128-146): standalone committed-git → default `SafeWorktree`; attached → `InPlace`; non-git/uncommitted → `ReadOnly`.
- **`trust.rs`** — `TrustKey{canonical_repo,command_digest}`(:20), `TrustStore`(:26): trust scoped to `(canonical repo path, SHA-256 of exact command set)`; any command change revokes trust; owner-only `trusted-commands.json`; fd-bound; corrupt/symlinked state **fails closed**.

None hit the network. **007.** **reference-only, with a loud caveat.** (a) The git worktree is **edit-isolation, not a process sandbox** — matches 007's "a worktree is NOT a sandbox"; pair Sandboy and treat `git.rs` as a reference for tamper-hardened checkout materialization. ## Control / data flow (load-bearing paths)

### Flow A — a WS `conduct` run: adapters → event → sessions → runner → server
```
browser ──WS {conduct,task,cwd}──▶ server::handle_session (server.rs:271)
  origin_allowed (:173)  ── cross-origin → 403
  cwd_within_root(cwd, launch_root) (:338)  ── outside → RunError, close
  active_run.try_acquire (:349)  ── busy → "run already active"
  spawn: OPERATOR_CONTROLS.scope(op, PROGRESS_SINK.scope(WsSink, run_conduct)) (:374)
    ▼ run_conduct (conduct.rs:226) ──per subtask──▶
      run_with_failover(worker ladder, write:true) (conduct.rs:449 → resilience.rs:190)
        └ run_to_completion (runner.rs:34)
            └ sessions::spawn (sessions.rs:76) ── FORK vendor CLI child (kill_on_drop)
                 reader: adapter.parse_line → AgentEvent → mpsc(256) (sessions.rs:116)
            collect loop: quota.record(Usage) (runner.rs:66); progress::emit(ev) (runner.rs:85)
    progress::emit ─▶ PROGRESS_SINK (task-local) ─▶ WsSink.on_event (server.rs:257)
       └ serialize AgentEvent ─▶ unbounded chan ─▶ writer task ─▶ WS text frame
    run finishes ─▶ ServerFrame::from(&ConductOutcome) (:415) ─▶ terminal frame ─▶ close
```
Cancellation: a `cancel` frame or socket-close in the `select!` loop (:388-407) → `run.abort()`; dropping the run future drops the `SessionHandle` → `ReaderTask::drop` aborts the reader → the `kill_on_drop` child is SIGKILLed **before the terminal frame** — a closed tab cannot keep burning quota.

### Flow B — how a subtask verdict is reached (the "DONE ≠ acceptance" spine)
```
worker runs (write:true) ─▶ worker_text (never accepted directly)
  capture_changes(cwd) = git diff HEAD (conduct.rs:515) ─ non-git → hard error
  run_verify(cwd) = sh -c build/test/lint (conduct.rs:521 → verify.rs:82)
  supervisor gate (advisory) (conduct.rs:538) ─ Halt → GLOBAL abort
  CONDUCTOR evaluation → EvalDecision{Accept|Rework|Fail} (conduct.rs:626)  ← primary (LLM) verdict
  GROUNDING VETO: if verify ran & failed & Accept → force Rework (conduct.rs:669-680)  ← mechanical
  on Accept → run_review_gate (conduct.rs:709 → :1056)
       review_blocks = verdict.has_critical(); unparseable = fail-CLOSED (:1112-1114)
       reworks remaining → Rework(findings); exhausted+blocking → ARBITER
       arbiter Ship → Ship; else/unparseable/unavailable → Fail (fail-closed :1216)
  completed.push(id)  OR  failed = Some(reason)  (first-wins) (conduct.rs:737 / :777)
```
`eval.rs` hardens this into the honesty keystone: discards the pipeline's self-report and scores the tree with an **independent external `run_verify`** after `restore_protected` undoes test-file tampering (`eval.rs:326-333`).

### Flow C — the conductor drives workers via the failover ladder (the choke point)
```
run_with_failover(ladder, label, build_req, quota, health, timeout) (resilience.rs:190)
  for rung in ladder:
     skip if health.is_dead / is_cold (:211)
     run_to_completion(rung.adapter, build_req(model)) (:241) ── NO `?` (spawn err demotes)
     classify_attempt → None(Completed) | ModelUnavailable | RateLimited | Transient (:164)
        Completed          → return FailoverResult{model_used,...} (:273)
        ModelUnavailable   → health.mark_dead; demote to ladder[i+1] (:284)  ← SILENT-ISH SWITCH
        RateLimited/Transient → same-rung retry×max_retries backoff; then demote (:251)
     every demotion: eprintln "↳ fell back A → B" + push Fallback (:304)  ← loud/audited
  all rungs exhausted → bail!("all N model rungs failed") (:313)  ← fail-closed
```
`ModelHealth` is shared across all roles in a run, so a model that dies in planning is skipped everywhere. **This is precisely the mechanism o7d's `ModelPolicy::Exact` must forbid** — to port, use a 1-rung ladder and treat any `Fallback` as a run-failing event.

## Coupling summary (what drags what if extracted)
- **Universal leaves (loose, safe to lift):** `event.rs`, `tokenizer.rs`, `confine.rs`, `catalog.rs`, `json_extract.rs`, `topology.rs`, `stagnation.rs`, `progress.rs` — pure/near-pure.
- **The run stack (one tight cluster):** `sessions → runner → resilience → conduct` + helpers (`routing`,`roles`,`changes`,`verify`,`review`,`operator`,`prompts`,`council`). `conduct.rs` is the hub; extracting it pulls the entire orchestrator + `quota`,`config`,`adapters`,`event`; `verify.rs` also reaches into `safety::commands`.
- **Front doors:** `server.rs` and `mcp.rs` each pull `conduct`/`resilience`/`council`/`review`/`verify`/`roles`/`quota`/`confine` + transport; `server.rs` adds `operator`+`progress`+`protocol`; `mcp.rs` adds `transcript`+`changes`.
- **Config/registry cluster (loose, read-only):** `config ← {models,recommend,wizard,protocol,doctor,auth}`; `catalog ← {models,recommend,auth}`.
- **Persistence:** `quota.rs` (SQLite `~/.consilium/usage.db`, the only DB, append-only token log) + `transcript.rs` (whole-file JSON, post-hoc). Both loosely coupled (only `event`/`serde`).
- **`safety/` (self-contained, mostly unwired):** internally cohesive (rustix + hardened git); only `commands.rs` wired (into `verify.rs`). The `git`/`trust`/`preflight`/`fs` subtree could be lifted wholesale — and *should be* if 007 wants the trust-store/worktree ideas, because Consilium itself doesn't route runs through them.

**Net 007 verdict:** adopt the *deterministic grounding oracle* (`verify.rs`) and *external-oracle scoring* (`eval.rs`) as the backbone of "o7d decides, DONE ≠ acceptance"; adopt the *process-lifecycle* discipline (`sessions.rs`/`runner.rs`) and the *append-only SQLite* shape (`quota.rs`) for the ledger; **reject** the *silent-failover* core (`resilience.rs`/`models.rs`) as incompatible with `ModelPolicy::Exact`; treat `safety/git.rs`+`trust.rs` as *reference-only* edit-isolation (a worktree is not Sandboy) whose unwired state is itself the warning; and note that neither the *transcript* (post-hoc, no recovery) nor the *operator* control (boundary-only, no mid-flight permission switch) yet meets the 007 append-only-ledger and live-permission-switch bars — both are adapt targets, not drop-ins.
